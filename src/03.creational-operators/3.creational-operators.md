# Creational Operators

Creational operators create new observables from plain values, collections, timers, and events. They are convenient because they let us describe the source of data without manually writing the same value generation logic every time.

We will implement these five basic operators:

1. `of(...values)`
2. `from(iterable)`
3. `range(start, count)`
4. `interval(period)`
5. `timer(dueTime)`

## `of(...values)`

`of` creates an observable that emits each provided value in order and then completes.

```typescript
function of<T>(...values: T[]): Observable<T> {
  return new Observable((observer) => {
    values.forEach((value) => observer.next(value));
    observer.complete();
  });
}
```

Example usage:

```typescript
of(10, 20, 30).subscribe({
  next(value) {
    console.log('of value:', value);
  },
  error(err) {
    console.error('of error:', err);
  },
  complete() {
    console.log('of complete');
  },
});
```

## `from(iterable)`

`from` creates an observable from any iterable value such as an array, string, `Set`, or generator.

```typescript
function from<T>(iterable: Iterable<T>): Observable<T> {
  return new Observable((observer) => {
    for (const item of iterable) {
      observer.next(item);
    }
    observer.complete();
  });
}
```

Example usage:

```typescript
import { from } from './creational-operators.js';

from(['A', 'B', 'C']).subscribe({
  next(value) {
    console.log('from value:', value);
  },
  error(err) {
    console.error('from error:', err);
  },
  complete() {
    console.log('from complete');
  },
});
```

## `range(start, count)`

`range` emits a fixed sequence of numbers starting at `start` and continuing for `count` emissions.

```typescript
export function range(start: number, count: number): Observable<number> {
  return new Observable<number>((observer) => {
    for (let index = 0; index < count; index += 1) {
      observer.next(start + index);
    }

    observer.complete();
  });
}
```

Example usage:

```typescript
import { range } from './creational-operators.js';

range(5, 4).subscribe({
  next(value) {
    console.log('range value:', value);
  },
  error(err) {
    console.error('range error:', err);
  },
  complete() {
    console.log('range complete');
  },
});
```

## `interval(period)`

`interval` emits incrementing numbers forever, waiting `period` milliseconds between each emission. Since it does not complete by itself, it returns cleanup logic that clears the timer when the subscription is unsubscribed.

```typescript
export function interval(period: number): Observable<number> {
  return new Observable<number>((observer) => {
    let value = 0;
    const intervalId = setInterval(() => {
      observer.next(value);
      value += 1;
    }, period);

    return () => clearInterval(intervalId);
  });
}
```

Example usage:

```typescript
import { interval } from './creational-operators.js';

const intervalSubscription = interval(1000).subscribe({
  next(value) {
    console.log('interval value:', value);
  },
  error(err) {
    console.error('interval error:', err);
  },
  complete() {
    console.log('interval complete');
  },
});

setTimeout(() => {
  intervalSubscription.unsubscribe();
  console.log('interval unsubscribed');
}, 3500);
```

## `timer(dueTime)`

`timer` waits for `dueTime` milliseconds, emits `0`, and then completes.

```typescript
export function timer(dueTime: number): Observable<number> {
  return new Observable<number>((observer) => {
    const timeoutId = setTimeout(() => {
      observer.next(0);
      observer.complete();
    }, dueTime);

    return () => clearTimeout(timeoutId);
  });
}
```

Example usage:

```typescript
import { timer } from './creational-operators.js';

timer(2000).subscribe({
  next(value) {
    console.log('timer value:', value);
  },
  error(err) {
    console.error('timer error:', err);
  },
  complete() {
    console.log('timer complete');
  },
});
```
