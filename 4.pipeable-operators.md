## Pipeable operator

A pipeable operator is a function that takes an observable as input and returns a new observable as output.

Internally, it subscribes to the source observable, applies some logic to the values it receives, and then emits the resulting values through the new observable.

## The `double` operator

Consider an observable that emits five numbers. We can create a function called `double` that takes this observable as input and returns another observable that emits doubled values.

```typescript
function double(input:Observable<number>){
    const outputObservable = new Observable<number>((subscriber) => {
        const subscription = input.subscribe({
            next(value) => {
                subscriber.next(value * 2);
            }
        })
        return () => subscription.unsubscribe();
    })
    return outputObservable
}
```

````typescript
## Both of the above are transform operators

Both of these operators transform the values emitted by the source observable and then emit the transformed values through a new observable. This is similar to the `map` method on arrays, which applies a transformation to each item and returns a new array.

```typescript
export type Operator<T, U> = (source: Observable<T>) => Observable<U>;

function map<T, U>(transform: (value: T) => U): Operator<T, U> {
  return (input: Observable<T>) => {
    const outputObservable = new Observable<U>((subscriber) => {
      const subscription = input.subscribe({
        next(value) {
          try {
            const transformedValue = transform(value);
            subscriber.next(transformedValue);
          } catch (err) {
            subscriber.error(err);
          }
        },
        error(err) {
          subscriber.error(err);
        },
        complete() {
          subscriber.complete();
        },
      });
      return () => subscription.unsubscribe();
    });
    return outputObservable;
  };
}

const numbers$ = of(1,2,3,4,5)
const squaredNumbers$ = map(value => value * value)(numbers$)
squaredNumbers$.subscribe({
    next(value) => console.log(value); // gets called with 1, 4, 9,16, 25
})
````

Since this `map` function is a higher-order function, it produces the operator we want when called with a transform function. For that reason, it is also called an operator factory function.

## first

This is a function that takes an observable, subscribes to it, returns an observable, that produces only the first value of the source observable. The first observable is unsubscribed as soon as its first value is read.

## Piping

Operator functions can be easily piped or composed as all of them take an Observable as input and produce another observable as output. We have already seen `pipe` function that can take a sequence of functions and can produce the final piped function. We can achieve similar piping by implementing a pipe method right inside `Observable`

```typescript
class Observable<T> {
  // --- other implementation
  pipe(...operators: Operator<unknown, unknown>[]): Observable<unknown> {
    return operators.reduce(
      (
        prevObservable: Observable<unknown>,
        currentOp: Operator<unknown, unknown>,
      ) => {
        return currentOp(prevObservable);
      },
      this,
    );
  }
}

//
const numbers$ = of(1, 2, 3, 4, 5);
const doubleSquared$ = numbers$.pipe(
  map((v) => 2 * v),
  map((v) => v * v),
);
```
